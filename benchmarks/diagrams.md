# Seahorse benchmark — diagrams

All diagrams are **design/architecture** (Mermaid). Any diagram showing measured *numbers* is
generated by `score.py` from real run data (`results/summary.mmd`) — none are hand-drawn, so the
repo never ships fabricated results.

## 1. The four conditions

```mermaid
flowchart LR
    T([SWE-bench instance<br/>problem_statement @ base_commit])
    T --> A[fable solo<br/>claude-fable-5]
    T --> B[opus solo<br/>claude-opus-4-8]
    T --> C[sonnet solo<br/>claude-sonnet-5]
    T --> D[seahorse<br/>advisor loop]

    subgraph solo [solo conditions - identical task string]
        A
        B
        C
    end

    subgraph orch [seahorse condition - same task, orchestration wrapper]
        D --> ADV{{advisor · Fable<br/>decompose + route}}
        ADV -->|mechanical chunk| BL[builder-light · Sonnet]
        ADV -->|subtle chunk| BH[builder-heavy · Opus]
        BL --> INT[integrate + verify]
        BH --> INT
    end

    A --> P[(git diff → model_patch)]
    B --> P
    C --> P
    INT --> P
```

## 2. Generation → eval → score pipeline

```mermaid
flowchart TD
    I[instances.txt<br/>pinned id + short/long] --> R[run.py]
    DS[(SWE-bench_Verified<br/>HF dataset)] --> R
    R -->|git worktree @ base_commit| W[repo checkout]
    W -->|claude -p --output-format json| CJ[result JSON<br/>cost · tokens · session]
    W -->|git diff --cached| PATCH[model_patch]
    CJ --> RJ[results/&lt;cond&gt;/&lt;id&gt;.json]
    PATCH --> RJ
    RJ --> PRED[results/&lt;cond&gt;/predictions.jsonl]
    PRED -->|python -m swebench.harness.run_evaluation<br/>Docker| EVAL[results/&lt;cond&gt;/eval.json<br/>resolved_ids]
    RJ --> S[score.py]
    EVAL --> S
    S --> OUT[summary.md · summary.csv · summary.mmd]
```

## 3. Metric capture per run

```mermaid
sequenceDiagram
    participant Run as run.py
    participant Git as git worktree
    participant CC as claude -p (headless)
    Run->>Git: worktree add --detach @ base_commit
    Run->>Run: t0 = monotonic()
    Run->>CC: prompt + --model + --output-format json
    CC-->>Run: {total_cost_usd, usage.*, session_id, result}
    Run->>Run: wall_ms = monotonic() - t0
    Run->>Git: add -A ; diff --cached  → model_patch
    Run->>Run: write results/<cond>/<id>.json
    Run->>Git: worktree remove --force
```

## 4. Stratification decision

```mermaid
flowchart TD
    X[instance] --> Q{difficulty annotated?}
    Q -->|yes| D{difficulty}
    D -->|"< 15 min fix"| SH[short]
    D -->|"15 min - 1 hour"| SH
    D -->|"1-4 hours"| LO[long]
    D -->|"> 4 hours"| LO

    PS -->|small: 1 file / few lines| SH
    PS -->|large: multi-file / many lines| LO

```

## 5. Cost-cap control flow

```mermaid
flowchart TD
    START[run.py] --> LOOP{next instance × condition}
    LOOP -->|result exists| SKIP[skip - idempotent] --> LOOP
    LOOP -->|cumulative ≥ --max-usd| STOP[stop + write predictions]
    LOOP -->|else| GEN[generate + capture cost]
    GEN --> ACC[spent += total_cost_usd] --> LOOP
```

## 6. Results chart (generated, not hand-drawn)

`score.py` writes `results/summary.mmd` — a Mermaid `xychart-beta` of mean cost per condition,
built **only** from real captured runs. Until a run exists it is intentionally absent. Example of
the *shape* it produces (values are placeholders, not measurements):

```mermaid
xychart-beta
    title "Mean cost per instance (USD) — SHAPE ONLY, populated by score.py"
    x-axis ["fable", "opus", "sonnet", "seahorse"]
    y-axis "USD" 0 --> 1
    bar [0, 0, 0, 0]
```
